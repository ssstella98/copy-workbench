## 1. 总则

### 1.1 设计原则

1. **清晰优先** — 用户应能在 3 秒内理解文案含义
2. **友好但不幼稚** — 迪士尼品牌的温暖感，但保持专业克制
3. **一致** — 同类场景、同类句式，同一概念、同一用词
4. **包容** — 不预设用户性别、年龄、能力；正确使用"国家及地区"
5. **合规** — 遵守《中华人民共和国广告法》及中国大陆互联网内容管理规定

### 1.2 语言

- 中文：简体中文（zh-CN），面向中国大陆用户
- 英文：国际英语（en-US），面向外国游客及系统默认语言
- 日文：计划支持（ja-JP）

## 2. 按文案类型的写作规范

### 2.1 Button（按钮）

在 166,616 条数据中占 **10.8%**，是最常见的文案类型之一。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 2-6 字（平均 5 字） | 1-3 词（平均 2.2 词） |
| 句式 | 动词/动词短语 | Imperative verb phrase |
| 语气 | 直接、可行动 | Direct, actionable |
| 标点 | 无句号 | No period |

**规范：**

- 按钮文案必须是一个**可立即理解的动作**
- 中文：动词 + 名词，如"确认下单"、"取消订单"、"添加乘客"
- 英文：Verb + Noun，如 "Place Order"、"Cancel"、"Add Guest"
- 避免模糊词："好的"、"知道了"只在轻量确认场景使用
- 避免超过 6 个中文字或 3 个英文单词

**范例：**

| ✅ 推荐 | ❌ 避免 |
|---------|--------|
| 确认下单 / Place Order | 点击这里确认 / Click Here |
| 取消订单 / Cancel Order | 不，我不想买了 / No |
| 去支付 / Pay Now | 前往支付页面进行支付操作 |

### 2.2 Error / 错误提示

占 **7.8%**。用户在遇到问题时看到的文案。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 10-30 字（平均 21 字） | 5-15 词（平均 9.6 词） |
| 句式 | 问题描述 + 建议操作 | Problem + suggested action |
| 语气 | 抱歉但不卑微 | Apologetic but professional |
| 标点 | 句号结尾 | Period |

**规范：**

- 结构：**发生了什么 + 用户可以做什么**
- 中文开头："抱歉，" / "很抱歉，" / 直接描述问题
- 英文开头：避免 "Error!" 开头，用 "Sorry," 或直接描述
- 技术细节不暴露给用户（如 HTTP 状态码）
- 提供明确的下一步动作

**范例：**

| ✅ 推荐 | ❌ 避免 |
|---------|--------|
| 改期失败，请稍后再试。 | Error 510999 |
| 抱歉，该优惠券已达到使用上限。请选择其他优惠券。 | 优惠券用不了 |
| Sorry, this coupon has reached its usage limit. Please select another. | Coupon error! |

### 2.3 Title / 标题

占 **15.3%**。页面、区块、卡片标题。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 3-12 字（平均 8 字） | 2-6 词（平均 4.1 词） |
| 句式 | 名词短语 | Noun phrase |
| 语气 | 中性、信息性 | Neutral, informative |
| 标点 | 无 | 无 |

**规范：**

- 中文：名词或名词短语，不加"的"/"了"/"吗"
- 英文：Title Case or Sentence case（保持一致）
- 页面主标题可较长，卡片/区块标题尽量短
- 不做完整句子

**范例：**

| ✅ 推荐 | ❌ 避免 |
|---------|--------|
| 我的订单 / My Orders | 您所有的订单都在这里 |
| 价格详情 / Price Details | 查看您的价格详情 |
| 预订成功 / Booking Confirmed | 恭喜！预订成功啦！ |

### 2.4 Modal / 弹窗

占 **4.1%**。平均最长——中文 97 字，英文 41 词。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 50-150 字 | 20-50 词 |
| 句式 | 背景说明 + 影响 + 选择引导 | Context + impact + choice |
| 语气 | 解释性、引导性 | Explanatory, guiding |
| 标点 | 句号结尾 | Period |

**规范：**

- 结构三段式：
  1. **标题**：用一句话概括弹窗目的
  2. **正文**：解释发生了什么 / 用户需要知道什么
  3. **按钮**：清晰的两个选择（确认 / 取消 或 去处理 / 稍后）
- 正文避免超过 3 句话
- 金额、日期等动态信息用变量：`{amount}`、`{date}`
- 涉及扣款/取消等敏感操作必须明确告知后果

**范例：**

```
标题：取消订单
正文：按照酒店取消规则，现在取消该订单将收取首晚房费（包含服务费和税费）。
      确认联系预订服务中心协助取消订单吗？
按钮：[我再想想] [确认取消]
```

### 2.5 Toast / 轻提示

占 **2.4%**。中文平均 39 字，英文 22 词。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 15-50 字 | 8-25 词 |
| 句式 | 状态 + 补充说明 | Status + additional info |
| 语气 | 简短友好 | Brief, friendly |
| 标点 | 视情况 | 视情况 |

**规范：**

- 成功 Toast：简洁，不加"成功"（系统已通过绿色图标传达）
  - "已添加到购物车" 而非 "添加购物车成功！"
- 失败 Toast：告知失败原因 + 建议
  - "网络连接失败，请检查网络后重试"
- 中性 Toast：状态告知，不加情绪
  - "未完成支付" 而非 "您还没付钱呢！"
- 最长不超过 50 中文字

### 2.6 Label / 标签文本

占 **6.1%**。输入框旁说明、字段名。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 3-12 字（平均 9 字） | 2-5 词（平均 4.2 词） |
| 句式 | 名词/名词短语 | Noun phrase |
| 语气 | 中性 | Neutral |
| 标点 | 无句号；冒号用于分隔 | 无 period；colon for separation |

**规范：**

- 字段名 + 说明的格式：`字段名：说明文字`
- 含变量时用 `{n}` 而非 `{number}`（中文场景）
- 参考编号等次要信息可用灰色文字降级

### 2.7 Link / 超链接

占 **3.0%**。中文平均 32 字。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 5-25 字 | 3-8 词 |
| 句式 | 动词/动词短语 | Action phrase |
| 语气 | 引导性 | Guiding |

**规范：**

- 中文：动词 + 名词，如"查看条款及细则"
- 英文：Verb + Noun，如 "View Terms and Conditions"
- 链接文案应描述**跳转后的内容**，而非"点击这里"
- 金额相关链接应展示优惠额度：`"使用优惠券，预计最高优惠 ¥{discountAmount}"`

### 2.8 Confirm / 确认状态

占 **3.4%**。操作完成后的反馈文案。

| 维度 | 中文 | English |
|------|------|---------|
| 长度 | 4-15 字（平均 26 字） | 2-5 词（平均 6.4 词） |
| 句式 | 状态描述 | Status statement |

**规范：**

- 成功：简洁状态词 + 可选补充
  - "兑换成功"、"已移除所选商品"
- 等待中：明确告知状态 + 预计时间
  - "等待支付"、"处理中，预计 5 分钟内完成"
- 失败：见 Error 规范

## 3. 通用写作规则

### 3.1 句式

| 规则 | 中文 | English |
|------|------|---------|
| 主动语态 | 用"我们已收到您的订单"而非"您的订单已被接收" | Use "We received your order" not "Your order has been received" |
| 第二人称 | 用"您"（不用"你"） | Use "You" |
| 动词优先 | 按钮/链接以动词开头 | Start buttons with verbs |
| 正面表述 | 用"请稍后再试"而非"不要现在放弃" | Use "Please try again" not "Don't give up" |

### 3.2 用词

**统一术语（部分）：**

| 概念 | 中文固定用词 | English fixed term |
|------|-------------|-------------------|
| 乐园 | 乐园 | Park |
| 酒店 | 酒店 | Hotel |
| 门票 | 门票 | Ticket |
| 优惠券 | 优惠券 | Coupon |
| 订单 | 订单 | Order |
| 游客 | 游客 | Guest |
| 预订 | 预订 | Book / Booking |
| 退款 | 退款 | Refund |
| 年卡 | 年卡 | Annual Pass |

**禁用词（广告法 + 内部规范）：**

- 禁用绝对化用语："最好"、"第一"、"唯一"、"国家级"、"顶级"、"极致"
- 政治敏感：统一使用"中国内地/中国大陆"，地图展示需合规
- 涉及台湾、香港、澳门时使用"中国台湾"、"中国香港"、"中国澳门"
- "国家及地区"（非"国家"）当涉及下拉选择时

### 3.3 标点符号

| 规则 | 中文 | English |
|------|------|---------|
| 句号 | 完整句用"。"，短标签/按钮不用 | Complete sentences use "." |
| 逗号 | 中文用"，"，不用"," | English uses "," |
| 冒号 | 字段名后用"：" | Colon ":" after labels |
| 省略号 | 用"……"（两个字符） | Use "..." |
| 感叹号 | 禁用。用文字传达重要性 | Avoid. Use words. |

### 3.4 数字与格式

- 金额：中文用 `¥1,239.00`，英文用 `¥1,239.00`
- 日期：中文用 `2024年8月16日`，英文用 `August 16, 2024`
- 电话：用 `+86 21 XXXX XXXX` 格式
- 百分比：用 `10%` 而非 `10 percent`（中文场景）

### 3.5 变量占位符

当前发现两种命名风格并存：
- `{camelCase}` — 推荐
- `{dot.notation}` — 历史遗留

**统一规范：**

- 变量名用 camelCase：`{couponName}`、`{discountAmount}`
- 通用变量：`{n}` 表示数量、`{amount}` 表示金额、`{date}` 表示日期
- 变量周围加空格（中文）：`最高优惠 ¥{discountAmount}`
- 变量不加空格（英文）：`save up to ¥{discountAmount}`

## 4. 合规规范（Compliance & Experience Rules）

> 基于 `compliance-experience-rules.json`（15 条规则），覆盖法律合规、品牌规范、内容安全、运营流程

### 4.1 国家/地区表述（HIGH · COMP-EXP-1.1）

- 下拉选择场景必须使用"国家及地区"，不得仅使用"国家"
- 涉及台湾、香港、澳门时正确使用"中国台湾"、"中国香港"、"中国澳门"
- 地图展示需经合规审核
- 监管依据：网信办《网络信息内容生态治理规定》、市场监管总局广告法

### 4.2 广告法禁用词（HIGH · COMP-EXP-1.2）

**绝对禁止使用的词汇：**

| 类别 | 禁用词 |
|------|--------|
| 绝对化用语 | 最、第一、唯一、国家级、世界级、顶级、最佳、最好、最高 |
| 夸张描述 | 极致、极致体验、绝对 |
| 保证承诺 | 100%、零风险、包治、保过 |

- 监管依据：《中华人民共和国广告法》第 9 条
- 适用阶段：发布前

### 4.3 行业特殊规范（MEDIUM · COMP-EXP-1.3）

- 金融类文案不得承诺保本保收益
- 医疗类文案不得承诺治愈率
- 教育类文案不得承诺升学率
- 监管依据：行业监管条例（银保监、药监局、教育部等）

### 4.4 跨境数据合规（MEDIUM · COMP-EXP-1.4）

- 涉及个人信息（PII）或重要数据跨境传输的文案需安全评估
- 监管依据：《数据安全法》《个人信息保护法》（PIPL）

### 4.5 品牌视觉规范（HIGH · COMP-EXP-2.1）

- Logo 使用、色彩、字体、版式需符合品牌指南
- 监管依据：企业内部品牌规范手册

### 4.6 版权素材（MEDIUM · COMP-EXP-2.2）

- 图片、字体、音乐、视频需获得授权或为原创
- 监管依据：《著作权法》、第三方版权库

### 4.7 商标使用（MEDIUM · COMP-EXP-2.3）

- 正确使用注册商标 ® / ™ 标记
- 不得侵犯第三方商标权
- 监管依据：《商标法》、企业商标库

### 4.8 敏感信息保护（HIGH · COMP-EXP-3.1）

- 文案中不得出现身份证号、手机号、银行卡号、密钥等真实敏感数据
- 示例数据必须明显为虚构（如 13800000000）
- 监管依据：《网络安全法》《个人信息保护法》

### 4.9 内容安全（HIGH · COMP-EXP-3.2）

- 禁止出现暴恐、色情、违禁品、违法活动等不良信息
- 监管依据：网信办"清朗"专项行动要求

### 4.10 隐私声明一致性（LOW · COMP-EXP-3.3）

- 文案中的数据使用描述需与隐私政策一致
- 监管依据：GDPR、PIPL 透明度原则

### 4.11 地域文化适配（MEDIUM · COMP-EXP-4.1）

- 内容需符合当地文化、宗教、习俗（如斋月、节日）
- 监管依据：本地化运营规范

### 4.12 渠道格式规范（LOW · COMP-EXP-4.2）

- 各平台发布需遵循对应的格式规范
- 监管依据：平台运营规则（微信、抖音等）

### 4.13 活动促销合规（MEDIUM · COMP-EXP-4.3）

- 促销规则需明确、抽奖需公正、不得误导用户
- 文案需包含活动条款链接或摘要
- 监管依据：《反不正当竞争法》《规范促销行为暂行规定》

### 4.14 多部门审批流（HIGH · COMP-EXP-5.1）

- 所有文案在发布前必须经过：法务 → 品牌 → PR（按需）
- 监管依据：企业内部控制制度

### 4.15 版本与归档（LOW · COMP-EXP-5.2）

- 保留历史版本，按规定归档/销毁
- 监管依据：企业档案管理规范、审计要求

## 5. 敏感场景特殊处理

### 5.1 支付与退款

- 涉及金额必须明确显示币种和精确数字
- 扣款前必须告知用户金额和不可逆性
- 退款文案必须告知预计到账时间
- 禁止暗示退款是"赠送"或"奖励"

### 5.2 个人信息收集

- 收集前必须告知用途
- 使用"国家及地区"而非"国家"
- 身份证/护照信息标注"加密传输，仅用于入园核验"

### 5.3 未成年人

- 涉及 16 岁以下用户必须提示监护人陪同要求
- 不直接向未成年人收集个人信息

### 5.4 健康与安全

- 游乐设施限制文案必须明确身高/年龄/健康状况要求
- 使用"为了您的安全"而非"禁止"
- 紧急情况文案需简洁、可操作

## 5. 质量检查清单

在提交文案审核前，逐项确认：

- [ ] Button 文案 ≤ 6 中文字 / 3 英文词
- [ ] Error 包含"发生了什么 + 怎么做"
- [ ] Modal 正文 ≤ 3 句
- [ ] 无广告法禁用词（"最好"、"第一"等）
- [ ] "国家及地区"替代"国家"（如下拉选择场景）
- [ ] 变量用 `{camelCase}` 格式
- [ ] 中文用全角标点，英文用半角标点
- [ ] 无感叹号
- [ ] 金额带币种符号
- [ ] 涉及扣款有明确金额和后果告知
