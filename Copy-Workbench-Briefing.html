Part 2
卡点 & 需要解决的问题
Producer 在文案管理上花费的时间中，超过 60% 是在做"整理"而非"创作"。
PAIN 01

Key 值整理混乱、依赖人工

虽然有一套固定的 Key 命名规则，但仍需 Producer 手动对照 Figma 设计图逐层整理页面层级，再逐个字段赋 Key 值。各 Studio 开发团队有各自的工作习惯，缺乏统一标准。

投入大易出错标准不统一
PAIN 02

开发交付 Key 文档过晚

开发通常在 Sprint 接近完成时才做 Knowledge Transfer，而 PM 在每个 Sprint 中持续对 Key 值进行增删改。Producer 拿到最终 Key 表时，留给写文案的时间已被严重压缩。

时间被压缩返工频繁
PAIN 03

AI 辅助效率低、无复用机制

已在用 AI 起草文案，但是逐条、逐场景与 AI 对话，效率低。各项目之间缺乏通用的术语库，Producer 仍靠人工检索去确认用词是否一致。

重复劳动术语不统一
PAIN 04

文档输出完全手工作业

Producer 手动将 UI 截图 + Key 值 + 中英日三语文案拼合为 Word 文档。一个中等规模项目（~200 字段）大约需要 2-3 个工作日仅用于排版和格式调整。

耗时 2-3 天格式易错版本混乱
Part 3
新的文案工作台计划
一个全新的独立应用，让 Producer 在 Design Review 通过后即可介入，
从 Figma 出发，一站式完成字段识别、Key 管理、多语言编辑和文档导出。

🔴 现状：分散线下

Figma 设计稿 → 人工逐字段整理 → Excel → 逐条 AI 对话 → 手动拼 Word → 多次格式转换

→

🟢 目标：一站闭环

上传 Figma → 自动识别字段 → 自动生成 Key → 集中编辑三语 Copy → 一键导出 Word + JSON

🔗

Figma 智能识别

连接 Figma 文件，自动扫描全量 TEXT 节点并生成字段清单。自动从 Frame 命名中提取页面层级结构，Producer 确认即可，无需逐字段手动整理。

自动识别 层级提取 智能去重
🔑

Key 值自动生成 & 校验

按照统一命名规则 {project}.{flow}.{page}.{function}.{desc} 自动为每个字段分配 Key。实时格式校验，不合规无法保存，确保 100% 命名规范。

自动生成 实时校验 批量修改
🌐

多语言文案集中编辑

每个字段在同一界面维护中/英/日三语 Copy。支持字段类型标签（Label / Error / Toast…），根据类型自动提示场景 Checklist，避免遗漏边界情况。

中/英/日 场景覆盖 批量导入
📄

一键导出 Word 文档

按页面层级自动排版为"左 UI 截图 + 右文案表格"的标准 Word 文档，可直接送审。彻底消除 2-3 天的手动排版工作。

Word 左图右表 按需导出
📦

结构化 JSON 导出

按标准 Schema 导出 JSON，直接交付开发导入项目。支持 Key 引用关系（ref），开发侧去重渲染。

JSON 标准 Schema Key 引用
👥

多人协作 & 项目管理

所有 Producer 共用同一个工作台，以项目维度隔离。多人并行编辑互不干扰，字段级别编辑锁防止冲突。项目状态一目了然。

项目隔离 编辑锁 状态管理
